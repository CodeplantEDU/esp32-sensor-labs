# 1️⃣ 디스플레이를 배우기 전에: 통신 방식부터 이해하기

디스플레이는 혼자 동작하지 않습니다.  
ESP32와 같은 마이크로컨트롤러가 “어떤 글자를 표시할지”, “어디에 점을 찍을지”, “언제 화면을 갱신할지”를 계속 보내줘야 합니다.  
이때 사용하는 데이터 전달 방식이 바로 **통신 방식**입니다.

대표적으로 교육용 실습에서 자주 만나는 통신 방식은 다음 3가지입니다.

- UART
- I2C
- SPI

이 세 방식은 모두 “장치끼리 데이터를 주고받는 방법”이지만, 사용하는 선의 수, 속도, 구조, 활용 목적이 서로 다릅니다.

---

## 1-1. UART 통신이란?

UART는 **Universal Asynchronous Receiver/Transmitter**의 약자입니다.  
우리말로는 보통 “범용 비동기 직렬 통신”이라고 부릅니다.

### 핵심 특징
- **TX(송신), RX(수신)** 두 선을 중심으로 통신
- **클럭선 없이** 약속된 속도(baud rate)로 데이터를 주고받음
- 구조가 단순해서 시리얼 모니터, 센서 통신, 모듈 디버깅에 많이 사용

### 왜 비동기일까?
UART는 송신기와 수신기가 같은 클럭선을 공유하지 않습니다.  
대신 서로 미리 “우리는 9600bps로 통신하자”, “우리는 115200bps로 통신하자”처럼 약속을 해 두고 데이터를 주고받습니다.  
그래서 **비동기(asynchronous)** 라고 부릅니다.

### 장점
- 구조가 단순해서 이해하기 쉽다
- 디버깅용으로 매우 자주 사용된다
- USB-시리얼 통신도 기본적으로 UART 개념과 연결된다

### 단점
- 여러 장치를 한 번에 연결하기에 불리하다
- 속도와 안정성 면에서 I2C, SPI보다 제한적일 수 있다

### 교육 현장에서의 예
- ESP32와 PC가 USB를 통해 시리얼 출력값을 주고받는 것
- GPS 모듈, 블루투스 모듈 일부와의 통신

---

## 1-2. I2C 통신이란?

I2C는 **Inter-Integrated Circuit**의 약자입니다.  
센서, LCD 모듈, OLED 모듈 등 여러 부품을 적은 핀으로 연결하고 싶을 때 매우 많이 사용합니다.

### 핵심 특징
- **SDA(데이터)**, **SCL(클럭)** 두 선만 사용
- 하나의 마스터(예: ESP32)가 여러 슬레이브 장치와 통신 가능
- 각 장치는 **주소(Address)** 를 가짐
- 핀을 적게 써서 배선이 간단함

### 동작 개념
ESP32가 “0x27 주소를 가진 장치야, 데이터를 받아라”라고 보내면  
그 주소를 가진 장치만 응답합니다.

즉, 같은 SDA/SCL 선을 여러 장치가 함께 써도  
주소가 다르면 서로 구별할 수 있습니다.

### 장점
- 핀 2개만 사용하므로 배선이 매우 간단하다
- 여러 장치를 같은 버스에 연결할 수 있다
- 교육용 실습에서 다루기 쉽다

### 단점
- SPI보다 일반적으로 속도가 느리다
- 주소 충돌이 생기면 문제가 발생할 수 있다
- 풀업 저항 등 회로 개념을 이해하면 더 좋다

### 교육 현장에서의 예
- I2C LCD 모듈
- SSD1306 OLED 모듈
- 온습도 센서, 가속도 센서, RTC 모듈 등

---

## 1-3. SPI 통신이란?

SPI는 **Serial Peripheral Interface**의 약자입니다.  
I2C보다 빠르게 데이터를 주고받아야 할 때 자주 사용합니다.

### 핵심 특징
- 보통 **MOSI, MISO, SCK, CS** 4개의 핵심 선 사용
- 클럭을 기준으로 빠르게 데이터 송수신
- 디스플레이, SD카드, 고속 센서 등에 많이 사용

### 각 선의 의미
- **MOSI**: 마스터 → 슬레이브 데이터
- **MISO**: 슬레이브 → 마스터 데이터
- **SCK**: 클럭
- **CS(또는 SS)**: 어떤 장치와 통신할지 선택

### 장점
- 속도가 빠르다
- 구조가 직관적이다
- 대용량 데이터 전송에 유리하다

### 단점
- I2C보다 핀이 많이 필요하다
- 장치 수가 늘수록 CS 핀이 추가로 필요하다
- 초보자 입장에서는 I2C보다 배선이 복잡하게 느껴질 수 있다

### 교육 현장에서의 예
- 고속 OLED/TFT 디스플레이
- SD 카드 모듈
- 일부 ADC/DAC 모듈

---

## 1-4. UART / I2C / SPI 비교

| 항목 | UART | I2C | SPI |
|------|------|-----|-----|
| 선 수 | 보통 2개(TX, RX) | 2개(SDA, SCL) | 보통 4개 이상 |
| 클럭 사용 | 없음 | 있음 | 있음 |
| 여러 장치 연결 | 불리함 | 쉬움(주소 사용) | 가능하지만 CS 필요 |
| 속도 | 보통 | 보통 | 빠름 |
| 배선 난이도 | 쉬움 | 쉬움 | 조금 복잡 |
| 교육용 활용 | 시리얼 출력, 통신 모듈 | 센서, LCD, OLED | 고속 디스플레이, SD카드 |

### 한 줄 정리
- **UART**: 간단한 1:1 통신
- **I2C**: 핀을 아끼며 여러 장치를 연결할 때 좋음
- **SPI**: 빠른 통신이 필요할 때 좋음

---

# 2️⃣ LCD란 무엇인가?

LCD는 **Liquid Crystal Display**, 즉 액정 디스플레이입니다.  
교육용 실습에서는 보통 **16x2 문자형 LCD**를 가장 많이 사용합니다.

여기서 16x2는 다음 뜻입니다.

- 한 줄에 16글자 표시 가능
- 총 2줄 표시 가능

즉, 간단한 문자 메시지나 센서값을 표시하기에 적합합니다.

### LCD의 특징
- 주로 **문자 출력**에 강함
- 구조가 비교적 단순함
- 가격이 저렴하고 수업용으로 활용하기 좋음
- 백라이트가 있어서 글자가 보임
- 그래픽 표현은 제한적임

### LCD는 왜 문자형이라고 할까?
보통 16x2 LCD는 “문자를 표시하기 위해 설계된 모듈”입니다.  
물론 사용자 정의 문자 기능도 일부 있지만,  
OLED처럼 자유롭게 점을 찍어서 복잡한 그림을 그리는 용도에는 적합하지 않습니다.

---

# 3️⃣ LCD의 기본 핀 구성 이해하기

일반적인 문자형 LCD(예: 1602 LCD)는 보통 **16개의 핀**을 가집니다.  
이 핀들은 전원, 제어, 데이터, 백라이트 역할로 나뉩니다.

아래 핀 설명은 실습에서 매우 중요합니다.

---

## 3-1. LCD 핀 전체 표

| 핀 이름 | 설명 | 핵심 역할 |
|--------|------|----------|
| GND | 접지 | 전류의 기준점 |
| VDD | 전원 | LCD 구동 전원 공급 |
| VO | 명암 조절 | 글자 진하기 조정 |
| RS | Register Select | 명령인지 데이터인지 구분 |
| RW | Read/Write | 읽기/쓰기 선택 |
| E | Enable | 데이터 입력 시점 확정 |
| D0~D7 | Data Bus | 실제 데이터 전달 |
| BLA | Back Light Anode | 백라이트 + |
| BLK | Back Light Cathode | 백라이트 - |

---

## 3-2. GND
**GND**는 접지입니다.  
모든 회로는 기준 전위가 있어야 안정적으로 동작하는데, 그 기준점이 GND입니다.

### 이해 포인트
- ESP32의 GND와 LCD의 GND는 반드시 연결되어야 함
- 전원이 공급되어도 GND가 공통이 아니면 신호 기준이 달라져 오동작 가능

---

## 3-3. VDD
**VDD**는 LCD에 전원을 공급하는 핀입니다.  
보통 5V를 사용합니다.

### 이해 포인트
- LCD는 내부 회로를 동작시키기 위해 전원이 필요함
- 데이터 신호만 연결한다고 동작하지 않음
- 모듈에 따라 3.3V 호환 여부가 다를 수 있으므로 데이터시트 확인이 좋음

---

## 3-4. VO
**VO**는 글자의 진하기(명암비)를 조절하는 핀입니다.

### 왜 필요한가?
LCD는 액정이 빛을 차단하는 정도에 따라 글자가 보이는데,  
명암 설정이 맞지 않으면 다음과 같은 현상이 나타납니다.

- 글자가 거의 안 보임
- 검은 네모만 보임
- 화면이 비정상적으로 흐릿함

### 보통 어떻게 연결하나?
- **가변저항(보통 10kΩ)** 가운데 핀을 VO에 연결
- 양 끝을 VDD와 GND에 연결
- 가변저항을 돌리며 적절한 명암을 맞춤

### 수업에서 자주 나오는 질문
“전원이 제대로 들어왔는데 왜 글자가 안 나와요?”  
→ VO 조절이 안 맞아서 그럴 가능성이 매우 큽니다.

---

## 3-5. RS
**RS(Register Select)** 는 LCD가 받는 값이  
“명령(command)”인지, “문자 데이터(data)”인지 구분하는 핀입니다.

### 쉽게 말하면
- **RS = 0** → LCD 설정 명령
- **RS = 1** → 표시할 문자 데이터

### 예시
- 커서를 어디로 이동할지
- 화면을 지울지
- 어떤 글자를 표시할지

이런 서로 다른 종류의 정보를 LCD가 구분해야 하므로 RS 핀이 필요합니다.

---

## 3-6. RW
**RW(Read/Write)** 는 읽기/쓰기 모드를 선택하는 핀입니다.

### 쉽게 말하면
- **RW = 0** → 쓰기
- **RW = 1** → 읽기

### 그런데 왜 보통 GND에 연결할까?
교육용 실습에서는 LCD로부터 값을 “읽어오는” 경우가 거의 없습니다.  
대부분 ESP32가 LCD에 글자만 보내면 되므로,  
RW를 그냥 **GND에 연결하여 항상 쓰기 모드**로 두는 경우가 많습니다.

### 장점
- 회로가 단순해짐
- 제어가 쉬워짐

---

## 3-7. E
**E(Enable)** 는 데이터 전송을 확정하는 핀입니다.

### 역할
ESP32가 RS와 데이터 핀에 값을 준비해놓고  
E 핀을 특정 타이밍으로 변화시키면  
LCD는 “아, 지금 데이터가 들어왔구나” 하고 받아들입니다.

즉, E는 일종의 **입력 확정 스위치** 역할을 합니다.

---

## 3-8. D0 ~ D7
이 핀들은 실제 데이터가 이동하는 **데이터 버스**입니다.

### 8비트 모드
- D0부터 D7까지 모두 사용
- 한 번에 8비트 전송

### 4비트 모드
- 보통 D4~D7만 사용
- 데이터를 나누어 전송
- 핀을 아낄 수 있음

### 수업 포인트
LCD를 직접 병렬 방식으로 연결하면 핀이 많이 필요합니다.  
그래서 실습에서는 보통 **I2C 변환 보드(PCF8574T)** 를 붙여서 사용합니다.

---

## 3-9. BLA / BLK
이 두 핀은 LCD 백라이트와 관련 있습니다.

- **BLA**: 백라이트 +
- **BLK**: 백라이트 -

### 역할
LCD는 자체 발광이 아니므로,  
글자가 잘 보이게 하려면 뒤에서 빛을 비춰주는 백라이트가 필요합니다.

### 이해 포인트
- 글자가 표시되는 원리와 백라이트는 구분해야 함
- 백라이트는 “보이게 해주는 조명” 역할
- 실제 문자는 액정의 배열 변화로 나타남

---

# 4️⃣ LCD를 직접 연결하면 왜 불편할까?

기본 LCD는 핀이 많습니다.  
전원 핀, 제어 핀, 데이터 핀까지 연결하려면 배선이 복잡해지고,  
ESP32의 GPIO도 많이 사용하게 됩니다.

예를 들어 병렬 방식으로 연결하면 보통 다음 문제가 생깁니다.

- 점퍼선이 많아져 헷갈림
- 오배선 가능성이 커짐
- GPIO 낭비가 큼
- 초보자에게 부담이 됨

이 문제를 해결하기 위해 많이 사용하는 것이 **PCF8574T 기반 I2C LCD 모듈**입니다.

---

# 5️⃣ PCF8574T 보드 설명

## 5-1. PCF8574T란?
PCF8574T는 **I/O 확장 칩**입니다.  
쉽게 말하면, I2C 신호를 받아서 여러 개의 디지털 출력처럼 활용할 수 있게 해주는 부품입니다.

LCD 뒤에 붙어 있는 I2C 백팩 모듈은  
이 칩을 이용해 LCD의 여러 핀을 간접적으로 제어합니다.

즉,

- ESP32는 SDA, SCL 두 선만 사용해서 명령을 보냄
- PCF8574T가 그 명령을 받아 LCD 핀들을 대신 제어함

그래서 복잡한 병렬 LCD가 **간단한 I2C LCD**처럼 보이게 됩니다.

---

## 5-2. PCF8574T를 쓰는 이유

### 1) 배선이 단순해진다
기존 LCD는 많은 핀이 필요하지만,  
I2C 변환 모듈을 사용하면 보통 다음 4개만 연결하면 됩니다.

- VCC
- GND
- SDA
- SCL

### 2) ESP32의 핀을 절약할 수 있다
디스플레이 하나 때문에 GPIO를 6개, 8개씩 쓰지 않아도 됩니다.

### 3) 수업용 실습에 유리하다
초보자도 배선 실수를 줄일 수 있고,  
회로가 단순해져 이해가 쉽습니다.

---

## 5-3. PCF8574T 보드의 주요 핀

| 핀 | 설명 |
|----|------|
| VCC | 전원 입력 |
| GND | 접지 |
| SDA | I2C 데이터 |
| SCL | I2C 클럭 |

### 추가로 알아둘 점
대부분의 I2C LCD 보드에는 주소를 바꿀 수 있도록  
A0, A1, A2 패드가 있는 경우가 많습니다.

이를 통해 여러 I2C 장치의 주소 충돌을 피할 수 있습니다.

---

## 5-4. I2C 주소란?

I2C 장치는 각자 주소를 가집니다.  
LCD용 PCF8574T 보드는 보통 다음 주소를 많이 사용합니다.

- `0x27`
- `0x3F`

하지만 제조사와 모듈에 따라 다를 수 있으므로  
실습 전에는 **I2C 스캐너 코드**로 실제 주소를 확인하는 것이 좋습니다.

---

## 5-5. LCD + PCF8574T의 교육적 의미
학생 입장에서 중요한 것은  
“LCD를 I2C로 쓰면 편하다”에서 끝나는 것이 아니라  
“원래는 병렬 방식으로 동작하는 LCD를, I/O 확장 칩이 대신 제어해주는 구조”라는 점을 이해하는 것입니다.

즉,  
**PCF8574T는 LCD의 본질을 바꾸는 것이 아니라, 연결 방식을 단순하게 만들어주는 중간 장치**입니다.

---

# 6️⃣ OLED란 무엇인가?

OLED는 **Organic Light Emitting Diode**의 약자입니다.  
유기 발광 다이오드를 이용한 디스플레이로, 각 픽셀이 스스로 빛을 냅니다.

### OLED의 큰 특징
- **백라이트가 필요 없음**
- 화면이 선명하고 대비가 좋음
- 문자뿐 아니라 그림, 아이콘, 선, 도형 표현 가능
- 소형 전자기기에서 많이 사용

### LCD와 가장 큰 차이
LCD는 뒤에서 빛을 비춰줘야 보입니다.  
OLED는 **픽셀 자체가 빛을 내기 때문에** 더 진한 검정색 표현과 선명한 화면이 가능합니다.

---

# 7️⃣ OLED 보드 설명

교육용 실습에서는 보통 **SSD1306 기반 128x64 OLED**를 많이 사용합니다.  
이 모듈은 I2C 방식과 SPI 방식이 모두 존재하지만,  
수업에서는 보통 **I2C 버전**을 가장 많이 사용합니다.

---

## 7-1. OLED I2C 핀 구성

| 핀 | 설명 |
|----|------|
| VCC | 전원 |
| GND | 접지 |
| SDA | I2C 데이터 |
| SCL | I2C 클럭 |

### 왜 이렇게 단순할까?
OLED 모듈 안에는 화면 제어용 드라이버 칩이 들어있고,  
이 칩이 I2C 명령을 받아 내부 화면 메모리를 제어합니다.  
그래서 외부에서는 4개 핀만으로도 충분히 제어할 수 있습니다.

---

## 7-2. OLED 주소
많은 SSD1306 I2C OLED의 기본 주소는 다음과 같습니다.

- `0x3C`

일부 모듈은 `0x3D`를 사용할 수 있으므로 역시 I2C 스캔이 중요합니다.

---

## 7-3. OLED가 실습에 좋은 이유

### 1) 문자 + 그래픽 모두 가능
단순 문자뿐 아니라
- 선
- 사각형
- 원(라이브러리에 따라)
- 아이콘
- 픽셀 단위 그래픽  
까지 표현할 수 있습니다.

### 2) 시각적으로 흥미롭다
학생들이 결과를 더 직관적으로 볼 수 있어  
실습 몰입도가 높습니다.

### 3) 센서 프로젝트와 잘 어울린다
온도, 습도, 거리, 조도 등 센서값을  
작은 대시보드처럼 보여주기 좋습니다.

---

# 8️⃣ LCD와 OLED의 차이 깊게 이해하기

| 항목 | LCD | OLED |
|------|-----|------|
| 표시 방식 | 액정이 빛을 조절 | 픽셀 자체 발광 |
| 백라이트 | 필요 | 불필요 |
| 주 용도 | 문자 표시 | 문자 + 그래픽 표시 |
| 화면 선명도 | 보통 | 높음 |
| 검정 표현 | 제한적 | 매우 우수 |
| 소비전력 특성 | 백라이트 영향 큼 | 화면 내용에 따라 달라짐 |
| 수업 난이도 | 구조 이해 쉬움 | 활용성이 높음 |

### 교육적으로 어떻게 설명하면 좋을까?
- **LCD**는 “문자를 간단하게 보여주는 표시장치”
- **OLED**는 “작은 화면에 텍스트와 그래픽을 자유롭게 보여주는 표시장치”

### 어떤 상황에서 어떤 것을 쓰나?
- 문자 몇 줄만 보여주면 된다 → **LCD**
- 센서값, 아이콘, 간단한 UI까지 표현하고 싶다 → **OLED**

---

# 9️⃣ ESP32와 연결하기

ESP32는 I2C 통신을 매우 편하게 사용할 수 있어서  
LCD(I2C), OLED(I2C) 실습에 잘 어울립니다.

보통 많이 사용하는 I2C 핀은 다음과 같습니다.

| 기능 | ESP32 GPIO |
|------|------------|
| SDA | GPIO 21 |
| SCL | GPIO 22 |

> 보드와 설정에 따라 다른 핀도 사용할 수 있지만,  
> 교육용 실습에서는 21(SDA), 22(SCL)을 많이 사용합니다.

---

## 9-1. 공통 연결 원칙
디스플레이가 LCD든 OLED든 I2C 버전이라면 연결 원칙은 거의 같습니다.

### 필수 연결
- VCC ↔ ESP32 전원
- GND ↔ ESP32 GND
- SDA ↔ ESP32 SDA
- SCL ↔ ESP32 SCL

### 주의할 점
- VCC 전압 범위를 모듈 사양에 맞춰야 함
- GND를 공통으로 묶어야 함
- I2C 주소를 확인해야 함
- 라이브러리에서 주소를 잘못 지정하면 화면이 안 나옴

# 1️⃣1️⃣ 실습 전에 꼭 해야 하는 것: I2C 주소 스캔

LCD든 OLED든 먼저 주소를 확인하는 습관이 중요합니다.

## I2C 스캔 코드
```python
import machine

# ESP32의 기본 I2C 핀 설정 (SDA=21, SCL=22)
i2c = machine.I2C(0, scl=machine.Pin(22), sda=machine.Pin(21))

print("I2C 스캐닝 시작...")
devices = i2c.scan()

if len(devices) == 0:
    print("연결된 I2C 장치가 없습니다. 선 연결을 확인하세요!")
else:
    for d in devices:
        print("장치 찾음! 주소: ", hex(d))

```

### 실행 결과 예시
- LCD가 연결되어 있으면 `0x27` 또는 `0x3F`
- OLED가 연결되어 있으면 `0x3C`

# 1️⃣2️⃣ LCD 실습

## 12-1. LCD 실습 목표
- I2C 방식 LCD를 ESP32와 연결할 수 있다
- LCD에 문자열을 출력할 수 있다
- 커서 위치를 바꾸어 여러 줄에 문자를 표시할 수 있다
- LCD가 문자형 디스플레이라는 점을 이해할 수 있다

---

## 12-2. 필요한 라이브러리
보통 아래 파일이 필요합니다.

- `lcd_api.py`
- `i2c_lcd.py`

이 파일들은 MicroPython용 LCD 제어 라이브러리입니다.

---

## 12-3. LCD 기본 출력 코드
```python
from machine import Pin, I2C
from i2c_lcd import I2cLcd
import time

# I2C 설정
i2c = I2C(0, scl=Pin(22), sda=Pin(21), freq=400000)

# LCD 주소와 크기 설정
LCD_ADDR = 0x27
lcd = I2cLcd(i2c, LCD_ADDR, 2, 16)

# 화면 지우기
lcd.clear()

# 첫 줄 출력
lcd.putstr("Hello, ESP32!")

time.sleep(2)

# 화면 지우고 새 메시지 표시
lcd.clear()
lcd.putstr("LCD Practice")
```

---

## 12-4. 코드 설명
### `I2C(...)`
ESP32의 I2C 통신을 설정합니다.  
여기서는 SDA=21, SCL=22를 사용했습니다.

### `I2cLcd(i2c, LCD_ADDR, 2, 16)`
I2C로 연결된 LCD를 초기화합니다.

- `i2c`: 사용할 I2C 객체
- `LCD_ADDR`: 장치 주소
- `2`: 줄 수
- `16`: 한 줄 글자 수

### `lcd.clear()`
기존 화면을 지웁니다.

### `lcd.putstr(...)`
문자열을 LCD에 출력합니다.

---

## 12-5. 두 줄 출력 예제
```python
from machine import Pin, I2C
from i2c_lcd import I2cLcd

i2c = I2C(0, scl=Pin(22), sda=Pin(21))
lcd = I2cLcd(i2c, 0x27, 2, 16)

lcd.clear()
lcd.move_to(0, 0)
lcd.putstr("Line 1: LCD")

lcd.move_to(0, 1)
lcd.putstr("Line 2: ESP32")
```

### 핵심 포인트
- `(0, 0)`은 첫 번째 줄 맨 앞
- `(0, 1)`은 두 번째 줄 맨 앞

---

## 12-6. 응용 예제: 카운터 표시
```python
from machine import Pin, I2C
from i2c_lcd import I2cLcd
import time

i2c = I2C(0, scl=Pin(22), sda=Pin(21))
lcd = I2cLcd(i2c, 0x27, 2, 16)

count = 0

while True:
    lcd.clear()
    lcd.move_to(0, 0)
    lcd.putstr("Counter")
    lcd.move_to(0, 1)
    lcd.putstr(str(count))
    count += 1
    time.sleep(1)
```

# 1️⃣3️⃣ OLED 실습

## 13-1. OLED 실습 목표
- SSD1306 OLED를 I2C 방식으로 연결할 수 있다
- 텍스트를 출력할 수 있다
- 픽셀 기반 디스플레이 개념을 이해할 수 있다
- 간단한 도형/선/화면 레이아웃을 구성할 수 있다

---

## 13-2. 필요한 라이브러리
- `ssd1306.py`

MicroPython 환경에서 SSD1306 OLED를 제어할 때 가장 많이 사용하는 라이브러리입니다.

---

## 13-3. OLED 기본 출력 코드
```python
from machine import Pin, I2C
import ssd1306

i2c = I2C(0, scl=Pin(22), sda=Pin(21), freq=400000)

oled = ssd1306.SSD1306_I2C(128, 64, i2c)

oled.fill(0)                 # 화면 지우기
oled.text("Hello OLED", 0, 0)
oled.text("ESP32 Practice", 0, 16)
oled.show()
```

---

## 13-4. 코드 설명

### `SSD1306_I2C(128, 64, i2c)`
- 가로 128픽셀
- 세로 64픽셀
- I2C 통신 사용

### `oled.fill(0)`
화면 전체를 0으로 채워 지웁니다.

### `oled.text("문자", x, y)`
특정 좌표에 텍스트를 표시합니다.

### `oled.show()`
버퍼에 준비한 내용을 실제 화면에 반영합니다.

> OLED는 보통 버퍼에 먼저 그린 뒤, `show()`를 호출해 화면을 한 번에 갱신합니다.

---

## 13-5. 도형 그리기 예제
```python
from machine import Pin, I2C
import ssd1306

i2c = I2C(0, scl=Pin(22), sda=Pin(21))
oled = ssd1306.SSD1306_I2C(128, 64, i2c)

oled.fill(0)
oled.rect(0, 0, 128, 64, 1)      # 테두리 사각형
oled.hline(0, 12, 128, 1)        # 가로선
oled.text("OLED Demo", 20, 2)
oled.text("Line / Box Test", 10, 24)
oled.show()
```

---

## 13-6. 응용 예제: 센서 대시보드 형태
```python
from machine import Pin, I2C
import ssd1306
import time

i2c = I2C(0, scl=Pin(22), sda=Pin(21))
oled = ssd1306.SSD1306_I2C(128, 64, i2c)

temperature = 24.5
humidity = 58

while True:
    oled.fill(0)
    oled.text("ENV MONITOR", 10, 0)
    oled.hline(0, 10, 128, 1)
    oled.text("Temp: {} C".format(temperature), 0, 20)
    oled.text("Hum : {} %".format(humidity), 0, 34)
    oled.show()
    time.sleep(1)
```

# 문제 해결(트러블슈팅)

## 화면이 안 나올 때 체크리스트

### 1) 전원 확인
- VCC와 GND가 제대로 연결되었는가?
- 모듈 전압 범위가 맞는가?

### 2) I2C 주소 확인
- `0x27`, `0x3F`, `0x3C` 중 실제 주소가 무엇인가?
- I2C 스캔 결과가 나오는가?

### 3) SDA / SCL 핀 확인
- SDA가 21, SCL이 22에 맞게 연결되었는가?
- 코드와 실제 배선이 일치하는가?

### 4) 라이브러리 확인
- `ssd1306.py`, `lcd_api.py`, `i2c_lcd.py` 파일이 보드에 업로드되어 있는가?
- 파일 이름이 정확한가?

### 5) LCD 명암 확인
- LCD인데 글자가 안 보인다면 VO 조절 문제일 수 있음
- 가변저항을 돌려 보며 확인
